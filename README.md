# Mini Banking System – Backend

A backend mini banking system built using Node.js, Express, and MongoDB.  
This project implements secure authentication, account management, and transaction handling with real-world business rules.

---

## Tech Stack

- Node.js
- Express.js
- MongoDB
- Mongoose
- JWT (JSON Web Tokens)
- bcrypt

---

## Features

### Authentication & Authorization
- User signup with hashed passwords
- User login with JWT-based authentication
- Protected routes using JWT middleware
- Get logged-in user details (`/me`)

### Accounts
- One user can have multiple bank accounts
- Account number generated on the backend
- Account limits:
  - Maximum 3 savings accounts
  - Maximum 1 current account

### Transactions
- Credit money to an account
- Debit money from an account
- Debit blocked if balance is insufficient
- Each balance change creates a transaction record
- View transactions:
  - Per account
  - Across all user accounts
- Pagination support for transaction history
- Strong ownership checks to prevent unauthorized access

---

## API Endpoints

### Auth
- `POST /api/auth/signup` – Register a new user
- `POST /api/auth/login` – Login and receive JWT
- `GET /api/auth/me` – Get logged-in user details

### Accounts
- `POST /api/accounts` – Create a new account (protected)

### Transactions
- `POST /api/transactions/credit` – Credit money (protected)
- `POST /api/transactions/debit` – Debit money (protected)
- `GET /api/transactions` – Get all transactions (protected, paginated)
- `GET /api/transactions/:accountId` – Get transactions of a specific account (protected)

---

## Business Rules

- Account numbers are generated by the backend
- A user can have:
  - Maximum 3 savings accounts
  - Maximum 1 current account
- Transaction amount must be greater than zero
- Debit is not allowed if account balance is insufficient
- Users can access only their own accounts and transactions
- All transactions are immutable
